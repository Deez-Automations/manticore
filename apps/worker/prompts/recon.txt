# EXECUTION PLAN — FOLLOW THESE STEPS IN ORDER

**Rules:**
- Use Grep, Glob, Read, Bash only. NO playwright. NO snapshots. NO browser.
- Complete each step fully before moving to the next.
- Skip external/ and vendor/ directories.
- Write JSON deliverable in exactly 3 Write/Edit calls as instructed in Step 3.

---

## STEP 1 — Grep for all endpoints and input vectors

Run these in sequence:

1. `Grep("function\s+\w+|Route\(|router\.|app\.get|app\.post|\$_GET|\$_POST|\$_REQUEST|\$_COOKIE", output_mode="files_with_matches")` — find all route/handler files
2. `Grep("\$_GET|\$_POST|\$_REQUEST|\$_COOKIE", output_mode="content")` — find all user input sources
3. `Grep("header\(|setcookie\(|session_start\(|session_destroy\(", type="php")` — find session/cookie management
4. `Grep("role|permission|admin|isAdmin|access_level", output_mode="files_with_matches")` — find authorization checks
5. `Grep("include\s*\(|require\s*\(|file_get_contents|fopen|curl_exec", type="php")` — find file/network ops

Read the top 3 most relevant files returned from these searches.

---

## STEP 2 — Read critical files from pre-recon

From the pre_recon_deliverable.md critical_files list, read any files not yet read. Maximum 3 additional files.

## STEP 3 — Write the deliverable as compact JSON

Write the deliverable in EXACTLY 3 Write/Edit calls:

**Call 1 — Create file with opening and first 3 keys:**
```
Write(".shannon/deliverables/recon_deliverable.md", content=<see schema below>)
```

**Call 2 — Append endpoints and input vectors:**
```
Edit(".shannon/deliverables/recon_deliverable.md", append endpoints section)
```

**Call 3 — Close the JSON:**
```
Edit(".shannon/deliverables/recon_deliverable.md", append closing brace)
```

### JSON Schema — keywords only, no sentences:

Use ONLY these keyword codes:
- Methods: `GET` `POST` `PUT` `DELETE`
- Auth flags: `no_csrf` `no_ratelimit` `no_auth` `session` `cookie` `md5` `plaintext`
- Input types: `GET` `POST` `COOKIE` `REQUEST` `SESSION` `HEADER` `FILE`
- Roles: `anon` `user` `admin`

```json
{
  "endpoints": [
    {"method": "GET", "path": "/vulnerabilities/sqli/", "file": "sqli/source/low.php", "auth": "session", "params": ["id"], "roles": ["user","admin"]},
    {"method": "POST", "path": "/login.php", "file": "dvwa/login.php", "auth": "no_auth", "params": ["username","password"], "flags": ["no_csrf","no_ratelimit"]}
  ],
  "input_vectors": [
    {"file": "sqli/source/low.php", "line": 12, "param": "id", "source": "GET", "sink": "mysql_query"},
    {"file": "exec/source/low.php", "line": 8, "param": "ip", "source": "POST", "sink": "exec"}
  ],
  "session_management": {"mechanism": "php_session", "flags": ["no_httponly","no_secure","no_samesite"]},
  "auth_flow": {"login_file": "dvwa/login.php", "password_hash": "md5", "flags": ["no_csrf","no_ratelimit"]},
  "roles": ["anon","user","admin"],
  "admin_paths": ["/dvwa/", "/setup.php"],
  "file_ops": [
    {"file": "fi/source/low.php", "line": 5, "func": "include", "source": "GET", "param": "page"}
  ],
  "network_ops": []
}
```

**RULES:**
- Include ALL findings — do not cap arrays
- Values must be keywords, file paths, line numbers, param names ONLY
- NEVER write a sentence or phrase as a value
- NO prose outside the JSON block
- Split across 3 Edit calls at natural key boundaries
